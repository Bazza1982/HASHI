# Auto Zeluda Evidence Schema

Every `auto_zeluda` run should keep an evidence pack with the following
artifacts where applicable.

## Run Charter

- `research_question`
- `sub_questions`
- `theory_framework`
- `dataset_boundary`
- `coding_unit`
- `round_plan`
- `exit_condition`

## Source Manifest

- `source_paths`
- `file_manifest`
- `source_type_map`
- `source_integrity_notes`
- `excluded_sources`

## Codebook Evidence

- `codebook_path`
- `version`
- `parent_codes`
- `child_codes`
- `definitions`
- `inclusion_rules`
- `exclusion_rules`
- `example_excerpts`
- `change_log`

## Coded Episode Table

Minimum fields:

- `excerpt_id`
- `source_file`
- `source_type`
- `date`
- `agent_or_persona`
- `task_context`
- `excerpt`
- `affect_type`
- `theory_code`
- `relational_code`
- `organisational_code`
- `governance_outcome`
- `analytic_memo`
- `coder`
- `round`
- `review_status`

## Review Evidence

- `reviewer`
- `reviewed_files`
- `source_context_checks`
- `coding_consistency_findings`
- `blockers`
- `non_blockers`
- `follow_ups`
- `sufficiency_judgement`

## Analysis Evidence

- `analytic_memo`
- `theme_candidates`
- `theme_evidence_matrix`
- `representative_excerpts`
- `negative_cases`
- `weak_themes`
- `writing_readiness_assessment`

## Closeout Evidence

- `final_codebook_path`
- `coded_dataset_path`
- `analysis_memo_paths`
- `theme_evidence_matrix_path`
- `review_report_path`
- `sufficiency_gate_result`
- `remaining_limitations`
- `recommended_findings_structure`
