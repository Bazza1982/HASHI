## Executive Summary

HASHI is currently Windows-primary. The codebase is already ~85% cross-platform but has four concrete blockers for macOS:

1. **TTS default** hard-coded to `"windows"` provider
2. **Workbench GPU detection** uses PowerShell / WMI
3. **`bridge-u.sh`** exists but is marked "macOS untested" — needs real testing + fixes
4. **No macOS installation documentation**

Everything else (locking, encoding, transport, adapters, onboarding) already works on Unix.

Both of these are **already correct** for macOS — `fcntl` is available natively on Darwin.

### Action Required

Create `orchestrator/platform_utils.py` as a central home for any future platform checks:

```python
# orchestrator/platform_utils.py
from __future__ import annotations
import sys

IS_WINDOWS = sys.platform == "win32"
IS_MACOS = sys.platform == "darwin"
IS_LINUX = sys.platform.startswith("linux")
IS_POSIX = not IS_WINDOWS


def default_tts_provider() -> str:
    """Return the best default TTS provider for this platform."""
    if IS_WINDOWS:
        return "windows"
    if IS_MACOS:
        return "macos"
    return "edge"


def open_url(url: str) -> None:
    """Open a URL in the default browser, cross-platform."""
    import subprocess
    if IS_WINDOWS:
        subprocess.Popen(["start", url], shell=True)
    elif IS_MACOS:
        subprocess.Popen(["open", url])
    else:
        subprocess.Popen(["xdg-open", url])
```

### Critical Considerations
- Do NOT use `os.name == "nt"` and `sys.platform == "win32"` interchangeably. On Cygwin, `os.name == "posix"` but `sys.platform == "cygwin"`. Prefer `sys.platform`.
- `pathlib.Path` already handles `/` vs `\` — never use string concatenation for paths.

#### B1 — `brew` / dependency check

Add macOS dependency pre-check at the top of `bridge-u.sh`:

```bash
# ── macOS dependency guard ──────────────────────────────────────────────────
if [[ "$(uname)" == "Darwin" ]]; then
    MISSING=()
    command -v python3  &>/dev/null || MISSING+=("python3 (brew install python@3.11)")
    command -v node     &>/dev/null || MISSING+=("node    (brew install node)")
    command -v ffmpeg   &>/dev/null || MISSING+=("ffmpeg  (brew install ffmpeg)")
    if [[ ${#MISSING[@]} -gt 0 ]]; then
        echo "❌  Missing dependencies on macOS:"
        printf '   • %s\n' "${MISSING[@]}"
        echo "   Install Homebrew first: https://brew.sh"
        exit 1
    fi
fi
```

#### B2 — `readlink -f` not available on macOS stock

macOS ships BSD `readlink`, which doesn't support `-f`. Fix:

```bash
# BEFORE (Linux-only)
SCRIPT_DIR="$(cd "$(dirname "$(readlink -f "$0")")" && pwd)"

# AFTER (cross-platform)
_resolve() {
    local src="$1"
    while [[ -L "$src" ]]; do
        local dir
        dir="$(cd -P "$(dirname "$src")" && pwd)"
        src="$(readlink "$src")"
        [[ "$src" != /* ]] && src="$dir/$src"
    done
    echo "$(cd -P "$(dirname "$src")" && pwd)/$(basename "$src")"
}
SCRIPT_DIR="$(cd "$(dirname "$(_resolve "$0")")" && pwd)"
```

#### B3 — `date --date` not available on macOS stock

macOS uses BSD `date`. Fix:

```bash
# BEFORE (GNU only)
date --date="@$EPOCH" '+%Y-%m-%d %H:%M:%S'

# AFTER (cross-platform) — use uname to detect platform
if [[ "$(uname)" == "Darwin" ]]; then
    date -r "$EPOCH" '+%Y-%m-%d %H:%M:%S'       # BSD / macOS
else
    date --date="@$EPOCH" '+%Y-%m-%d %H:%M:%S'  # GNU / Linux
fi
```

#### B4 — Workbench browser open

```bash
# BEFORE
start "http://localhost:$CLIENT_PORT"   # Windows
xdg-open "http://localhost:$CLIENT_PORT" # Linux

# AFTER
case "$(uname)" in
    Darwin)  open "http://localhost:$CLIENT_PORT" ;;
    Linux)   xdg-open "http://localhost:$CLIENT_PORT" ;;
    MINGW*)  start "http://localhost:$CLIENT_PORT" ;;
esac
```

### Critical Considerations
- macOS ships with `zsh` as default since Catalina. Users may have `/bin/bash` at version 3.2 (GPL2). Always use `#!/usr/bin/env bash` and test with bash 3.2 compatibility (`${var,,}` lowercase syntax is bash 4+).
- `brew` puts binaries in `/opt/homebrew/bin` (Apple Silicon) or `/usr/local/bin` (Intel). Ensure PATH includes both.

## Workflow D — Skills Cross-Platform Execution

### Current State

Skills are stored in `skills/` and executed as shell scripts. Currently only `.ps1` / `.bat` variants may exist.

### Action Required

Create a skill execution router in `orchestrator/skill_runner.py`:

```python
# orchestrator/skill_runner.py
from __future__ import annotations
import asyncio
import sys
from pathlib import Path


async def run_skill(skill_dir: Path, args: list[str] | None = None) -> tuple[int, str, str]:
    """
    Discover and run the best script variant for this platform.
    Priority:
      macOS/Linux : run.sh > run.py
      Windows     : run.ps1 > run.bat > run.py
    """
    args = args or []

    if sys.platform == "win32":
        candidates = [
            ("powershell", ["-NoProfile", "-File", str(skill_dir / "run.ps1")]),
            ("cmd",        ["/c", str(skill_dir / "run.bat")]),
        ]
    else:
        candidates = [
            ("bash",    [str(skill_dir / "run.sh")]),
        ]

    # Always allow pure-python fallback
    py_script = skill_dir / "run.py"

    for exe, cmd_args in candidates:
        script = Path(cmd_args[-1])
        if script.exists():
            proc = await asyncio.create_subprocess_exec(
                exe, *cmd_args, *args,
                stdout=asyncio.subprocess.PIPE,
                stderr=asyncio.subprocess.PIPE,
            )
            out, err = await proc.communicate()
            return proc.returncode, out.decode("utf-8", errors="replace"), err.decode("utf-8", errors="replace")

    if py_script.exists():
        proc = await asyncio.create_subprocess_exec(
            sys.executable, str(py_script), *args,
            stdout=asyncio.subprocess.PIPE,
            stderr=asyncio.subprocess.PIPE,
        )
        out, err = await proc.communicate()
        return proc.returncode, out.decode("utf-8", errors="replace"), err.decode("utf-8", errors="replace")

    raise FileNotFoundError(f"No runnable skill script found in {skill_dir}")
```

### Critical Considerations
- Shell scripts need `chmod +x` after git checkout on macOS/Linux. Add to `bridge-u.sh` post-clone setup: `find skills/ -name "run.sh" -exec chmod +x {} +`
- Line endings: `.sh` files committed from Windows may have CRLF. Add `.gitattributes`: `*.sh text eol=lf`

### E2 — Create macOS TTS Provider

Create `orchestrator/tts_providers/macos.py`:

```python
# orchestrator/tts_providers/macos.py
from __future__ import annotations
import asyncio
from pathlib import Path
from .base import BaseTTSProvider
from orchestrator.voice_synthesizer import VoiceAsset, convert_wav_to_ogg


# The macOS `say` command writes AIFF by default.
# Pass -o with a .wav extension and it writes WAVE (PCM).
# Rate: `say -r N` where N is words-per-minute. Default ~175 wpm.
# We map HASHI rate (-10 to +10) → wpm range (100–250).

_RATE_MIN = 100    # wpm at rate=-10
_RATE_MAX = 250    # wpm at rate=+10
_RATE_DEFAULT = 175


def _hashi_rate_to_wpm(rate: int) -> int:
    """Convert HASHI rate (-10..+10) to macOS say words-per-minute."""
    clamped = max(-10, min(10, rate))
    return int(_RATE_DEFAULT + clamped * (_RATE_MAX - _RATE_MIN) / 20)


class MacOSTTSProvider(BaseTTSProvider):
    provider_name = "macos"

    # Built-in macOS English voices as of Ventura/Sonoma.
    # Full list: `say -v ?`
    VOICES = {
        "Ava":      "en-US female (enhanced)",
        "Samantha": "en-US female",
        "Alex":     "en-US male",
        "Moira":    "en-IE female",
        "Karen":    "en-AU female",
        "Daniel":   "en-GB male",
        "Siri":     "Siri voice (if enabled in System Settings)",
    }

    async def synthesize(
        self,
        text: str,
        output_dir: Path,
        stem: str,
        voice_name: str | None = None,
        rate: int = 0,
        max_chars: int = 1200,
        provider_options: dict | None = None,
    ) -> VoiceAsset:
        spoken_text = self.prepare_spoken_text(text, max_chars=max_chars)
        if not spoken_text:
            raise RuntimeError("No spoken text after preparation.")

        output_dir.mkdir(parents=True, exist_ok=True)
        wav_path = output_dir / f"{stem}.wav"
        ogg_path = output_dir / f"{stem}.ogg"

        cmd = ["say"]
        if voice_name:
            cmd += ["-v", voice_name]
        if rate != 0:
            cmd += ["-r", str(_hashi_rate_to_wpm(rate))]
        cmd += ["-o", str(wav_path)]
        # Pass text as final argument (say does NOT read from stdin by default)
        cmd.append(spoken_text)

        proc = await asyncio.create_subprocess_exec(
            *cmd,
            stdout=asyncio.subprocess.PIPE,
            stderr=asyncio.subprocess.PIPE,
        )
        stdout, stderr = await proc.communicate()

        if proc.returncode != 0:
            err = stderr.decode("utf-8", errors="replace").strip()
            raise RuntimeError(
                f"macOS `say` failed (exit {proc.returncode}): {err or '(no stderr)'}"
            )
        if not wav_path.exists():
            raise RuntimeError(
                f"macOS `say` exited 0 but wav not created at {wav_path}"
            )

        await convert_wav_to_ogg(self.ffmpeg_cmd, wav_path, ogg_path)
        return VoiceAsset(
            provider=self.provider_name,
            text=text,
            spoken_text=spoken_text,
            wav_path=wav_path,
            ogg_path=ogg_path,
        )
```

### Complete Cross-Platform GPU Detection Replacement

```javascript
// workbench/server/index.js — replace getGpuInfo()

const { execSync } = require('child_process');
const os = require('os');

let gpuCache = { data: null, time: 0 };

function getGpuInfo() {
  const now = Date.now();
  if (gpuCache.data && now - gpuCache.time < 5000) return gpuCache.data;

  let result = { available: false, name: null, vendor: null, status: 'Unavailable',
                 npu: { available: false, name: null, status: 'Unavailable' } };

  try {
    const platform = process.platform;

    if (platform === 'win32') {
      const raw = execSync(
        `powershell -NoProfile -Command "` +
        `Get-CimInstance Win32_VideoController | ` +
        `Select-Object -First 1 Name,AdapterRAM,Status | ` +
        `ConvertTo-Json -Compress"`,
        { encoding: 'utf8', timeout: 5000, windowsHide: true, stdio: ['ignore','pipe','pipe'] }
      ).trim();
      const gpu = JSON.parse(raw);
      result = {
        available: true,
        name: gpu.Name || 'Unknown GPU',
        vendor: detectVendor(gpu.Name),
        status: gpu.Status || 'OK',
        npu: detectNpuWindows(gpu.Name),
      };

// Add this helper function at the end of the file:
function detectNpuWindows(gpuName) {
  // Detect NPU on Windows (AMD Ryzen AI, Intel NPU, Qualcomm)
  const name = (gpuName || '').toLowerCase();
  if (name.includes('ryzen') && name.includes('ai')) {
    return { available: true, name: 'AMD Ryzen AI NPU', status: 'Available' };
  }
  // Add more NPU detection as needed
  return { available: false, name: null, status: 'Unavailable' };
}

    } else if (platform === 'darwin') {
      // system_profiler returns YAML-like output; parse Chipset Model line
      const raw = execSync(
        `system_profiler SPDisplaysDataType 2>/dev/null`,
        { encoding: 'utf8', timeout: 8000, stdio: ['ignore','pipe','pipe'] }
      ).trim();

      const chipsetMatch = raw.match(/Chipset Model:\s*(.+)/);
      const vramMatch    = raw.match(/VRAM[^:]*:\s*(.+)/i);
      const name = chipsetMatch ? chipsetMatch[1].trim() : null;

      result = {
        available: !!name,
        name: name || 'Apple Integrated GPU',
        vendor: name?.includes('Apple') ? 'Apple' : detectVendor(name || ''),
        status: name ? 'OK' : 'Integrated',
        npu: {
          // Apple Silicon M1+ has Neural Engine; detect via sysctl
          available: isAppleSilicon(),
          name: isAppleSilicon() ? 'Apple Neural Engine' : null,
          status: isAppleSilicon() ? 'Available' : 'Unavailable',
        },
      };

    } else {
      // Linux: try nvidia-smi, then lspci, then /sys
      const name = linuxGpuName();
      result = {
        available: !!name,
        name: name || 'Unknown',
        vendor: detectVendor(name || ''),
        status: name ? 'OK' : 'Unavailable',
        npu: { available: false, name: null, status: 'Unavailable' },
      };
    }
  } catch (_) {
    // swallow — return default unavailable result
  }

  gpuCache.data = result;
  gpuCache.time = now;
  return result;
}

function isAppleSilicon() {
  try {
    const chip = execSync('sysctl -n machdep.cpu.brand_string 2>/dev/null', { encoding: 'utf8', timeout: 2000 }).trim();
    return chip.includes('Apple');
  } catch { return false; }
}

function linuxGpuName() {
  try {
    // nvidia-smi fastest path
    const raw = execSync('nvidia-smi --query-gpu=name --format=csv,noheader 2>/dev/null', { encoding: 'utf8', timeout: 3000 }).trim();
    if (raw) return raw.split('\n')[0];
  } catch {}
  try {
    const raw = execSync('lspci 2>/dev/null | grep -i vga', { encoding: 'utf8', timeout: 3000 }).trim();
    if (raw) return raw.split(':').pop()?.trim() || null;
  } catch {}
  return null;
}

function detectVendor(name) {
  const n = name.toLowerCase();
  if (n.includes('nvidia'))  return 'NVIDIA';
  if (n.includes('amd') || n.includes('radeon')) return 'AMD';
  if (n.includes('intel'))   return 'Intel';
  if (n.includes('apple'))   return 'Apple';
  return 'Unknown';
}
```

### System Info (CPU/RAM) — macOS Variants

```javascript
function getCpuInfo() {
  const platform = process.platform;
  if (platform === 'win32') {
    // existing PowerShell code
  } else if (platform === 'darwin') {
    try {
      const brand = execSync('sysctl -n machdep.cpu.brand_string 2>/dev/null', { encoding: 'utf8', timeout: 2000 }).trim();
      const cores = os.cpus().length;
      return { name: brand, cores, status: 'OK' };
    } catch {}
  }
  // Linux fallback
  return { name: os.cpus()[0]?.model || 'Unknown CPU', cores: os.cpus().length, status: 'OK' };
}
```

### Critical Considerations
- `system_profiler` on macOS can be slow (2–4s cold start). Cache results aggressively (5s TTL shown above).
- On macOS 14+ (Sonoma), `system_profiler` may require Full Disk Access for some data. `SPDisplaysDataType` does not require any special permission.
- Never use `execSync` without `timeout` — a hung GPU query will freeze the workbench API.

| Issue | Risk | Fix |
|-------|------|-----|
