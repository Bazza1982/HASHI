# Writing Template Validators

A generated writing template should pass these checks:

- It is preceded by deep reading notes that pass the reading gate.
- It states the article genre and journal/style context.
- It lists the full section hierarchy.
- It gives rough paragraph counts by section.
- It explains what each paragraph or paragraph cluster does.
- It uses placeholders rather than source claims.
- It does not copy long source phrases.
- It distinguishes structure from content.
- It includes language-style guidance.
- It includes scope limits.
- It is usable for writing a different paper in the same style.

For strict validation, compare the template against the source paragraph map
and check that the template keeps rhetorical roles while removing topic-specific
content.

## Deep reading gate checks

Before template generation, verify:

- `article_identity` is complete.
- `core_argument` is stated in transferable terms.
- `genre_and_structure` explains why the article is organised this way.
- `section_logic` covers every major section.
- `paragraph_role_map` covers paragraphs or paragraph clusters.
- `evidence_devices` explains the function of tables, figures, and appendices.
- `language_moves` records reusable writing moves.
- `transferable_structure` separates reusable form from source content.
- `quality_risks` records uncertainty instead of hiding it.
