# Deck Update Playbook

Use this playbook when Barry asks for a MoH data platform project update deck.

## Required Workflow

1. Read the source or previous-month deck.
2. Extract slide sequence, title pattern, section divider pattern, visual
   hierarchy, and recurring slide types.
3. Identify what changes for the new update period:
   - date
   - reporting month
   - key developments
   - timeline status
   - pipeline diagram updates
   - risks and forward plan
   - related business
   - upcoming meetings
4. Reuse the learned deck rhythm rather than inventing a new presentation.
5. Build the deck in PowerPoint.
6. Export to PDF for visual inspection.
7. Check slide-by-slide for layout consistency and executive readability.

## Slide Pattern

1. Cover
2. Blank/transition slide if present in source
3. Monthly Update divider
4. Key Developments
5. Project Timeline Overview
6. Project Timeline Overview continued
7. Data Pipeline
8. Key Risks and Forward Plan
9. Related Business divider
10. Related Business
11. Questions & Answers
12. Upcoming Meetings
13. Closing/back slide if present in source

## Quality Rules

- Keep text concise and boardroom-readable.
- Do not over-design.
- Preserve status-update logic before adding visual polish.
- Use tables for risk/forward-plan information.
- Use diagrams only when they clarify process or pipeline flow.
- Speaker notes are optional unless Barry asks for presenter rehearsal.
